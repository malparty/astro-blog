# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

Package manager is **pnpm** (v9.4.x in CI). Dev server runs through TinaCMS, which wraps `astro dev`.

- `pnpm dev` — TinaCMS + Astro dev server (use this, not `pnpm start`)
- `pnpm start` — Plain `astro dev` without Tina
- `pnpm build` — Astro build; `postbuild` runs `pagefind --site dist` to generate the static search index
- `pnpm preview` — Preview the production build
- `pnpm sync` — Regenerate Astro content collection types (run after editing `src/content/config.ts` or content schemas)
- `pnpm lint` — ESLint
- `pnpm format` / `pnpm format:check` — Prettier (uses `--plugin-search-dir=.` to pick up `prettier-plugin-astro`)

A Husky `pre-commit` hook runs `lint-staged`, which formats staged `.astro/.js/.ts/.md/.mdx/.json` with Prettier.

## Architecture

Astro 4 static blog with TinaCMS for content editing, Tailwind for styling, Pagefind for client-side search, and FTP deploy via GitHub Actions.

**Content pipeline.** Posts live as `.md` / `.mdx` in `src/content/blog/`. The schema in `src/content/config.ts` enforces frontmatter via Zod — notably `category` is `z.enum(CATEGORIES)`, so adding a post with a category not listed in `src/data/categories.ts` will fail the build. `heroImage` uses Astro's `image()` helper, so cover images must be relative paths resolvable from the post file (typically `../../assets/images/...`). Drafts (`draft: true`) are filtered out by `src/utils/post.ts#getPosts` but Astro itself is configured with `markdown.drafts: true` and `mdx.drafts: true` so drafts still render in dev.

**Routing.** All pages are in `src/pages/`. Dynamic routes: `post/[...slug].astro`, `category/[category]/[page].astro`, `tags/[...tag]/index.astro`. Pagination size comes from `src/data/site.config.ts#paginationSize`. RSS is generated by `src/pages/rss.xml.ts`.

**Post helpers.** `src/utils/post.ts` is the single source of truth for listing/filtering posts (excludes drafts, sorts by `pubDate` desc). Use it instead of calling `getCollection('blog')` directly so draft filtering stays consistent.

**Search.** Pagefind indexes `dist/` after build. There is no dev-time index — search only works against a built site (`pnpm build && pnpm preview`).

**TinaCMS.** `tina/config.ts` defines a parallel schema for the blog collection used by the Tina admin UI. **This schema must be kept in sync with `src/content/config.ts`** — Tina has its own field definitions and does not read the Zod schema. The Tina admin builds to `public/admin/` (gitignored) and is excluded from FTP uploads in `.github/workflows/deploy.yml`. `clientId` and `token` are `null` in committed config (local-only Tina); cloud Tina would require setting those.

**Site metadata.** `src/data/site.config.ts` holds author/title/description/locale/pagination. The deployed site URL is set in `astro.config.mjs` (`site: 'https://malparty.fr'`) — change both when forking.

**Deploy.** `.github/workflows/deploy.yml` runs on push to `main`: pnpm install → `pnpm build` → FTP upload of `./dist/` to `/www/` on the server. Credentials come from `FTP_SERVER`/`FTP_USERNAME`/`FTP_PASSWORD` GitHub secrets. `netlify.toml` exists but the active deploy path is FTP. The workflow sets `core.autocrlf false` because Windows line endings break the build.
